# ADR-0005: Public Analytics Dashboard тАФ `/stats` Page

## Status

Proposed тАФ 2026-03-07

## Context

Phase 1 of ADR-0004 is deployed. The blog collects page views in D1 and serves aggregate data via `GET /api/stats`. The next step is a public `/stats` page тАФ transparent analytics, visible to anyone, embodying the blog's "Prompted by human, generated by AI" ethos.

### What We Have

**API response** (`/api/stats?days=30`):
```json
{
  "period": { "start": "2026-02-05", "end": "2026-03-07" },
  "totals": { "views": 2, "visitors": 1, "ai_fetches": 0 },
  "by_path": [{ "path": "/the-agentic-product-engineer/", "views": 1, "visitors": 1 }],
  "by_country": [{ "country": "US", "views": 2 }],
  "by_day": [{ "date": "2026-03-07", "views": 2, "visitors": 1 }],
  "by_referrer": [{ "referrer": "github.com/gkoreli", "views": 1 }]
}
```

**Blog tech stack** (constraints):
- Static HTML generated at build time by `@nisli/core` pipeline
- Templates: TypeScript functions тЖТ `html` tagged template literals
- CSS custom properties: `--color-bg`, `--color-surface`, `--color-text`, `--color-link`, `--color-border`, `--color-text-muted`
- Dark/light mode: `data-theme` attribute on `<html>`, toggled by `<nisli-theme-toggle>` web component
- Layout: sidebar + main content, `--content-max: 800px`, mobile breakpoint at 768px (sidebar stacks on top)
- Client JS: esbuild-bundled ES modules, currently ~1.2KB (`main.js` = theme toggle only)
- esbuild entry points: `[CLIENT_ENTRY, STYLES_SRC]` тЖТ `main.js` + `main.css`
- Font: Lora (serif) тАФ editorial, literary aesthetic

### What the Open Source Projects Use (Verified in Source)

**Plausible** (21KтнР) тАФ `assets/js/dashboard/`:
- React + Chart.js (`chart.js v3.3.2`) + D3 (`d3 v7.9.0` for world map) + Tailwind CSS
- Dashboard sections: top stats bar, line graph, sources, pages, locations (world map), devices, behaviours
- Ranked lists use `assets/js/dashboard/stats/bar.js` тАФ a `<div>` with `width: ${(count / maxCount) * 100}%` and absolute positioning. Dead simple.
- `MAX_ITEMS = 9` per list, `ROW_HEIGHT = 32px`, `ROW_GAP = 4px`
- Public shared dashboards via `/share/:slug` тАФ same dashboard, read-only

**Counterscale** (2KтнР) тАФ `packages/server/app/routes/dashboard.tsx`:
- React + Recharts (`recharts v2.13`) + React Router (Remix)
- `ComposedChart` with `Area` + `Line` for views/visitors overlay
- Custom tooltip: date, visitors, views, bounce rate
- Admin-only behind `requireAuth()`

**Key insight**: Both use React + heavy charting because they're multi-site SaaS dashboards. We're building one transparent page on a personal blog. Different requirements тАФ but we should not compromise on UX quality.

### Per-Page Bundle Architecture (Verified with esbuild)

The blog is SSG тАФ each page is its own HTML file. The `/stats` page loads its own JS bundle that no other page loads. **Tested and confirmed** тАФ esbuild with 3 entry points (`main.ts`, `main.css`, `stats.ts`) produces 4 isolated output files:

```
Every page loads:        main.js (1.2KB) + main.css
Only /stats loads:       stats.js (52KB min / 23KB gz) + stats.css (1.7KB min / 768B gz)
```

- `stats.ts` imports `uplot` + `uplot/dist/uPlot.min.css` + `../styles/stats.css`
- esbuild merges all CSS imports into a single `stats.css` output
- No cross-contamination between main and stats bundles
- Navigate away from `/stats` тЖТ browser GCs the entire stats module

### What We Need to Visualize

| Section | Data source | Needs chart lib? | Interaction |
|---------|-------------|-------------------|-------------|
| Totals bar | `totals` | No тАФ 3 numbers | None |
| Daily trend | `by_day` (~30 pts) | **Yes** тАФ area + line | Hover tooltip, cursor crosshair |
| Top pages | `by_path` | No тАФ ranked list | None |
| Referrers | `by_referrer` | No тАФ ranked list | None |
| Countries | `by_country` | No тАФ ranked list | None |

Only the daily trend chart benefits from a charting library. Everything else is numbers and ranked lists with percentage-width bars.

## Proposals

### Proposal A: Hand-Rolled SVG + Custom Tooltip

SVG `<path>` for area, `<polyline>` for line, invisible `<rect>` overlays for hover, absolutely-positioned `<div>` tooltip.

- тЬЕ Zero deps, ~2-3KB total, SVG uses CSS `currentColor` (auto dark/light), printable, accessible (in DOM)
- тЭМ Must handle: empty data, single point, resize, axis labels, tooltip viewport boundaries, mobile touch
- тЭМ No cursor crosshair snap-to-point, no drag-to-zoom тАФ would take 200+ lines to approximate poorly

### Proposal B: uPlot тнР

uPlot (9.9KтнР, MIT, v1.6.32, last push Feb 2026) for daily trend. Everything else HTML/CSS.

- тЬЕ Built-in cursor crosshair with snap-to-nearest-point
- тЬЕ Built-in legend with live values on hover тАФ no custom tooltip code
- тЬЕ Drag-to-zoom for exploring date ranges
- тЬЕ Handles all edge cases: empty data, single point, gaps, resize, retina
- тЬЕ "No transitions or animations тАФ they're always pure distractions" (README) тАФ matches our aesthetic
- тЬЕ "Smooth spline interpolation... strongly discouraged: Your data is misrepresented!" тАФ evidence-based data viz philosophy
- тЬЕ 23KB gzip + 772B CSS тАФ smallest battle-tested time series library
- тЬЕ 166K data points in 25ms. Our 30-90 points are instant.
- тЭМ Canvas: doesn't use CSS vars (need JS bridge via `getComputedStyle()`)
- тЭМ Canvas: not in DOM (mitigate with `aria-label` on container)
- тЭМ Canvas: blank in print (mitigate with `@media print` text fallback)
- тЭМ Theme switch: destroy + recreate (~5 lines JS)

### Proposal C: Chart.js

- тЬЕ Most popular (66KтнР), Plausible uses it
- тЭМ 65KB gzip тАФ 2.8x larger than uPlot for the same result
- тЭМ Same Canvas limitations as uPlot
- тЭМ Designed for complex multi-dataset charts тАФ we'd disable most features

## Decision

**Proposal B (uPlot)** for the daily trend chart. Everything else is HTML/CSS.

### Why uPlot Over Hand-Rolled SVG

Zero-dependency where it makes sense тАФ not as dogma. We chose `crypto.subtle` over `@noble/hashes` because the platform provides a better native. We chose raw D1 over Drizzle because we have 2 queries. For interactive time series, the platform provides nothing. The cursor crosshair, snap-to-point, live legend, and drag-to-zoom are real UX features that would take 200+ lines of hand-rolled SVG to approximate worse.

The architectural guarantee: **uPlot loads only on `/stats`**. Separate esbuild entry point, separate `<script>` tag, separate CSS. Blog posts never load it.

### Why uPlot Over Chart.js

- 2.8x smaller (23KB vs 65KB)
- Philosophically aligned тАФ author explicitly rejects animations, stacked charts, spline interpolation
- Exact quote: "No transitions or animations тАФ they're always pure distractions."
- Less configuration surface = less code to write

## Anti-Patterns

1. **тЭМ Chart library in `main.js`**: Must be in separate `stats.js` entry point. Never in the shared bundle.
2. **тЭМ React for one page**: Counterscale uses React + Recharts (135KB+ gz). Adding React for a single page is architectural pollution.
3. **тЭМ World map for <30 countries**: Plausible uses D3 + topojson (~80KB) for a choropleth. A ranked list with flag emoji is better UX for small datasets and costs 0 bytes.
4. **тЭМ Animations on data charts**: uPlot's author and our aesthetic agree тАФ animations on data visualization are distractions.
5. **тЭМ Client-side aggregation**: All aggregation happens in D1 SQL. Client receives pre-aggregated JSON and renders. No sorting/filtering/grouping in browser.
6. **тЭМ CSS entry point for stats styles**: Don't add `stats.css` as a separate esbuild entry point тАФ it conflicts with CSS extracted from `stats.ts` imports. Import all CSS from `stats.ts` and let esbuild merge.
7. **тЭМ `<link>` and `<script>` in `<body>`**: While HTML5-valid, `<link rel="stylesheet">` in body risks FOUC. Use `pageShell`'s `head` param for correct `<head>` placement.
8. **тЭМ Dynamic `import()` for code splitting**: Unnecessary complexity. SSG pages control their own `<script>` tags тАФ that IS the code splitting. No router or framework needed.
9. **тЭМ Importing `@nisli/core` signals in `stats.ts`**: Stats module should be framework-independent. Use `MutationObserver` on `data-theme` attribute to detect theme changes тАФ decoupled from the signal system.

## Gotchas

1. **Canvas + dark mode**: uPlot uses Canvas API colors (strings), not CSS variables. Must read CSS vars via `getComputedStyle(document.documentElement)` and pass to uPlot config. On theme toggle, destroy and recreate. The theme toggle sets `data-theme` attribute тАФ detect with `MutationObserver(documentElement, { attributes: true, attributeFilter: ['data-theme'] })`.

2. **Canvas + print**: Canvas renders blank in `@media print`. Add print CSS that hides the canvas container and shows a text summary instead.

3. **Canvas + accessibility**: Canvas is opaque to screen readers. Add `role="img"` and `aria-label="Daily views: N total views, N visitors over the last 30 days"` on the chart container. The rest of the page (totals, ranked lists) is fully accessible HTML.

4. **Canvas + dark mode cursor points**: uPlot issue #780 тАФ cursor point fill defaults to white, looks wrong on dark backgrounds. Fix: explicitly set `cursor.points.fill` to read from series fill color. Verified fix in issue comments.

5. **uPlot responsive resize**: uPlot requires explicit `width`/`height` тАФ no auto-resize. Use `ResizeObserver` on the chart container, call `chart.setSize({ width: container.clientWidth, height: 300 })`. Use fixed height (300px) to avoid CSS layout issues (uPlot issue #1075 тАФ flexbox/grid height calculation bugs).

6. **uPlot data format**: Expects columnar arrays `[[timestamps], [views], [visitors]]`, not row objects. X-axis must be unix timestamps in **seconds** (not milliseconds). Transform: `new Date("2026-03-07").getTime() / 1000`. Arrays must have length тЙе 2.

7. **uPlot single data point**: Arrays must have length тЙе 2. If `by_day` has 0 or 1 entries, either skip the chart or pad with a zero-value entry for the previous day.

8. **Country flag emoji edge cases**: Cloudflare returns non-ISO 2-letter codes:
   - `T1` тЖТ Tor exit node тЖТ ЁЯФТ "Tor"
   - `XX` тЖТ Unknown тЖТ ЁЯМР "Unknown"
   - `A1` тЖТ Anonymous proxy тЖТ ЁЯФТ "Proxy"
   - `A2` тЖТ Satellite provider тЖТ ЁЯУб "Satellite"
   - `EU` тЖТ European Union тЖТ ЁЯЗкЁЯЗ║ (has a real flag emoji)
   - `AP` тЖТ Asia/Pacific тЖТ ЁЯМП "Asia-Pacific"
   - Standard codes: `String.fromCodePoint(...[...code].map(c => 0x1F1E6 + c.charCodeAt(0) - 65))`

9. **Empty state**: Blog is new тАФ periods may have zero data. Show friendly message, not broken charts or empty tables.

10. **Loading state (FOUC)**: Page loads as static HTML, then fetches `/api/stats`. Use CSS skeleton placeholders (animated gray bars matching final layout dimensions). Pure CSS `@keyframes` shimmer. Replaced by real content when data arrives. No layout shift (good CLS).

11. **esbuild CSS merge**: When `stats.ts` imports both `uplot/dist/uPlot.min.css` and `../styles/stats.css`, esbuild merges them into a single `stats.css` output (verified: 768 bytes gzip total). Don't also add `stats.css` as a separate entry point тАФ it would conflict.

## Best Practices

1. **тЬЕ Separate entry point per page type**: `main.ts` for all pages, `stats.ts` for `/stats` only. Templates control which `<script>` tags load. This IS code splitting for SSG тАФ no framework needed.

2. **тЬЕ CSS variables as the Canvas theme bridge**: Read `--color-link`, `--color-border`, `--color-text-muted`, `--color-bg` via `getComputedStyle()`. Pass as uPlot `stroke`, `fill`, axis `stroke`, grid `stroke`. On theme change, re-read and recreate.

3. **тЬЕ Percentage-width bars for ranked lists**: Same pattern as Plausible's `bar.js` тАФ `<div>` with `width: ${(value / maxValue) * 100}%` and `background: var(--color-surface)`. Pure CSS, no chart library.

4. **тЬЕ URL-based period selector**: `/stats?days=7`, `/stats?days=30`, `/stats?days=90`. Shareable, bookmarkable. Client JS reads `URLSearchParams`, passes to `/api/stats?days=N`. Default: 30.

5. **тЬЕ Country code тЖТ flag emoji with fallback map**: Standard ISO codes get flag emoji via `String.fromCodePoint()`. Non-standard codes (T1, XX, A1, A2, AP) get a lookup table with descriptive emoji + label.

6. **тЬЕ Skeleton loading state**: CSS-only `@keyframes` shimmer on placeholder elements matching final layout. Replaced when data arrives. No layout shift.

7. **тЬЕ `MutationObserver` for theme changes**: Decoupled from `@nisli/core` signals. Stats module watches `data-theme` attribute on `<html>`. Framework-independent.

8. **тЬЕ Fixed chart height, responsive width**: Height 300px (fixed). Width from `container.clientWidth` via `ResizeObserver`. Avoids CSS layout calculation bugs with uPlot.

9. **тЬЕ `pageShell` head param**: Add optional `head?: string` to `pageShell()` for per-page `<link>` and `<script>` tags in `<head>`. Correct HTML semantics, no FOUC, reusable for future pages.
