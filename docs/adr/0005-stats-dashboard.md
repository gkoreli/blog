# ADR-0005: Public Analytics Dashboard ‚Äî `/stats` Page

## Status

Proposed ‚Äî 2026-03-07

## Context

Phase 1 of ADR-0004 is deployed. The blog collects page views in D1 and serves aggregate data via `GET /api/stats`. The next step is a public `/stats` page ‚Äî transparent analytics, visible to anyone, embodying the blog's "Prompted by human, generated by AI" ethos.

### What We Have

**API response** (`/api/stats?days=30`):
```json
{
  "period": { "start": "2026-02-05", "end": "2026-03-07" },
  "totals": { "views": 2, "visitors": 1, "ai_fetches": 0 },
  "by_path": [{ "path": "/the-agentic-product-engineer/", "views": 1, "visitors": 1 }],
  "by_country": [{ "country": "US", "views": 2 }],
  "by_day": [{ "date": "2026-03-07", "views": 2, "visitors": 1 }],
  "by_referrer": [{ "referrer": "github.com/gkoreli", "views": 1 }]
}
```

**Blog tech stack** (constraints):
- Static HTML generated at build time by `@nisli/core` pipeline
- Templates: TypeScript functions ‚Üí `html` tagged template literals
- CSS custom properties: `--color-bg`, `--color-surface`, `--color-text`, `--color-link`, `--color-border`, `--color-text-muted`
- Dark/light mode: `data-theme` attribute on `<html>`, toggled by `<nisli-theme-toggle>` web component
- Layout: sidebar + main content, `--content-max: 800px`, mobile breakpoint at 768px (sidebar stacks on top)
- Client JS: esbuild-bundled ES modules, currently ~1.2KB (`main.js` = theme toggle only)
- esbuild entry points: `[CLIENT_ENTRY, STYLES_SRC]` ‚Üí `main.js` + `main.css`
- Font: Lora (serif) ‚Äî editorial, literary aesthetic

### What the Open Source Projects Use (Verified in Source)

**Plausible** (21K‚≠ê) ‚Äî `assets/js/dashboard/`:
- React + Chart.js (`chart.js v3.3.2`) + D3 (`d3 v7.9.0` for world map) + Tailwind CSS
- Dashboard sections: top stats bar, line graph, sources, pages, locations (world map), devices, behaviours
- Ranked lists use `assets/js/dashboard/stats/bar.js` ‚Äî a `<div>` with `width: ${(count / maxCount) * 100}%` and absolute positioning. Dead simple.
- `MAX_ITEMS = 9` per list, `ROW_HEIGHT = 32px`, `ROW_GAP = 4px`
- Public shared dashboards via `/share/:slug` ‚Äî same dashboard, read-only

**Counterscale** (2K‚≠ê) ‚Äî `packages/server/app/routes/dashboard.tsx`:
- React + Recharts (`recharts v2.13`) + React Router (Remix)
- `ComposedChart` with `Area` + `Line` for views/visitors overlay
- Custom tooltip: date, visitors, views, bounce rate
- Admin-only behind `requireAuth()`

**Key insight**: Both use React + heavy charting because they're multi-site SaaS dashboards. We're building one transparent page on a personal blog. Different requirements ‚Äî but we should not compromise on UX quality.

### Per-Page Bundle Architecture (Verified with esbuild)

The blog is SSG ‚Äî each page is its own HTML file. The `/stats` page loads its own JS bundle that no other page loads. **Tested and confirmed** ‚Äî esbuild with 3 entry points (`main.ts`, `main.css`, `stats.ts`) produces 4 isolated output files:

```
Every page loads:        main.js (1.2KB) + main.css
Only /stats loads:       stats.js (52KB min / 23KB gz) + stats.css (1.7KB min / 768B gz)
```

- `stats.ts` imports `uplot` + `uplot/dist/uPlot.min.css` + `../styles/stats.css`
- esbuild merges all CSS imports into a single `stats.css` output
- No cross-contamination between main and stats bundles
- Navigate away from `/stats` ‚Üí browser GCs the entire stats module

### What We Need to Visualize

| Section | Data source | Needs chart lib? | Interaction |
|---------|-------------|-------------------|-------------|
| Totals bar | `totals` | No ‚Äî 3 numbers | None |
| Daily trend | `by_day` (~30 pts) | **Yes** ‚Äî area + line | Hover tooltip, cursor crosshair |
| Top pages | `by_path` | No ‚Äî ranked list | None |
| Referrers | `by_referrer` | No ‚Äî ranked list | None |
| Countries | `by_country` | No ‚Äî ranked list | None |

Only the daily trend chart benefits from a charting library. Everything else is numbers and ranked lists with percentage-width bars.

## Proposals

### Proposal A: Hand-Rolled SVG + Custom Tooltip

SVG `<path>` for area, `<polyline>` for line, invisible `<rect>` overlays for hover, absolutely-positioned `<div>` tooltip.

- ‚úÖ Zero deps, ~2-3KB total, SVG uses CSS `currentColor` (auto dark/light), printable, accessible (in DOM)
- ‚ùå Must handle: empty data, single point, resize, axis labels, tooltip viewport boundaries, mobile touch
- ‚ùå No cursor crosshair snap-to-point, no drag-to-zoom ‚Äî would take 200+ lines to approximate poorly

### Proposal B: uPlot ‚≠ê

uPlot (9.9K‚≠ê, MIT, v1.6.32, last push Feb 2026) for daily trend. Everything else HTML/CSS.

- ‚úÖ Built-in cursor crosshair with snap-to-nearest-point
- ‚úÖ Built-in legend with live values on hover ‚Äî no custom tooltip code
- ‚úÖ Drag-to-zoom for exploring date ranges
- ‚úÖ Handles all edge cases: empty data, single point, gaps, resize, retina
- ‚úÖ "No transitions or animations ‚Äî they're always pure distractions" (README) ‚Äî matches our aesthetic
- ‚úÖ "Smooth spline interpolation... strongly discouraged: Your data is misrepresented!" ‚Äî evidence-based data viz philosophy
- ‚úÖ 23KB gzip + 772B CSS ‚Äî smallest battle-tested time series library
- ‚úÖ 166K data points in 25ms. Our 30-90 points are instant.
- ‚ùå Canvas: doesn't use CSS vars (need JS bridge via `getComputedStyle()`)
- ‚ùå Canvas: not in DOM (mitigate with `aria-label` on container)
- ‚ùå Canvas: blank in print (mitigate with `@media print` text fallback)
- ‚ùå Theme switch: destroy + recreate (~5 lines JS)

### Proposal C: Chart.js

- ‚úÖ Most popular (66K‚≠ê), Plausible uses it
- ‚ùå 65KB gzip ‚Äî 2.8x larger than uPlot for the same result
- ‚ùå Same Canvas limitations as uPlot
- ‚ùå Designed for complex multi-dataset charts ‚Äî we'd disable most features

## Decision

**Proposal B (uPlot)** for the daily trend chart. Everything else is HTML/CSS.

### Why uPlot Over Hand-Rolled SVG

Zero-dependency where it makes sense ‚Äî not as dogma. We chose `crypto.subtle` over `@noble/hashes` because the platform provides a better native. We chose raw D1 over Drizzle because we have 2 queries. For interactive time series, the platform provides nothing. The cursor crosshair, snap-to-point, live legend, and drag-to-zoom are real UX features that would take 200+ lines of hand-rolled SVG to approximate worse.

The architectural guarantee: **uPlot loads only on `/stats`**. Separate esbuild entry point, separate `<script>` tag, separate CSS. Blog posts never load it.

### Why uPlot Over Chart.js

- 2.8x smaller (23KB vs 65KB)
- Philosophically aligned ‚Äî author explicitly rejects animations, stacked charts, spline interpolation
- Exact quote: "No transitions or animations ‚Äî they're always pure distractions."
- Less configuration surface = less code to write

## Anti-Patterns

1. **‚ùå Chart library in `main.js`**: Must be in separate `stats.js` entry point. Never in the shared bundle.
2. **‚ùå React for one page**: Counterscale uses React + Recharts (135KB+ gz). Adding React for a single page is architectural pollution.
3. **‚ùå World map for <30 countries**: Plausible uses D3 + topojson (~80KB) for a choropleth. A ranked list with flag emoji is better UX for small datasets and costs 0 bytes.
4. **‚ùå Animations on data charts**: uPlot's author and our aesthetic agree ‚Äî animations on data visualization are distractions.
5. **‚ùå Client-side aggregation**: All aggregation happens in D1 SQL. Client receives pre-aggregated JSON and renders. No sorting/filtering/grouping in browser.
6. **‚ùå CSS entry point for stats styles**: Don't add `stats.css` as a separate esbuild entry point ‚Äî it conflicts with CSS extracted from `stats.ts` imports. Import all CSS from `stats.ts` and let esbuild merge.
7. **‚ùå `<link>` and `<script>` in `<body>`**: While HTML5-valid, `<link rel="stylesheet">` in body risks FOUC. Use `pageShell`'s `head` param for correct `<head>` placement.
8. **‚ùå Dynamic `import()` for code splitting**: Unnecessary complexity. SSG pages control their own `<script>` tags ‚Äî that IS the code splitting. No router or framework needed.
9. **‚ùå Importing `@nisli/core` signals in `stats.ts`**: Stats module should be framework-independent. Use `MutationObserver` on `data-theme` attribute to detect theme changes ‚Äî decoupled from the signal system.

## Gotchas

1. **Canvas + dark mode**: uPlot uses Canvas API colors (strings), not CSS variables. Must read CSS vars via `getComputedStyle(document.documentElement)` and pass to uPlot config. On theme toggle, destroy and recreate. The theme toggle sets `data-theme` attribute ‚Äî detect with `MutationObserver(documentElement, { attributes: true, attributeFilter: ['data-theme'] })`.

2. **Canvas + print**: Canvas renders blank in `@media print`. Add print CSS that hides the canvas container and shows a text summary instead.

3. **Canvas + accessibility**: Canvas is opaque to screen readers. Add `role="img"` and `aria-label="Daily views: N total views, N visitors over the last 30 days"` on the chart container. The rest of the page (totals, ranked lists) is fully accessible HTML.

4. **Canvas + dark mode cursor points**: uPlot issue #780 ‚Äî cursor point fill defaults to white, looks wrong on dark backgrounds. Fix: explicitly set `cursor.points.fill` to read from series fill color. Verified fix in issue comments.

5. **uPlot responsive resize**: uPlot requires explicit `width`/`height` ‚Äî no auto-resize. Use `ResizeObserver` on the chart container, call `chart.setSize({ width: container.clientWidth, height: 300 })`. Use fixed height (300px) to avoid CSS layout issues (uPlot issue #1075 ‚Äî flexbox/grid height calculation bugs).

6. **uPlot data format**: Expects columnar arrays `[[timestamps], [views], [visitors]]`, not row objects. X-axis must be unix timestamps in **seconds** (not milliseconds). Transform: `new Date("2026-03-07").getTime() / 1000`. Arrays must have length ‚â• 2.

7. **uPlot single data point**: Arrays must have length ‚â• 2. If `by_day` has 0 or 1 entries, either skip the chart or pad with a zero-value entry for the previous day.

8. **Country flag emoji edge cases**: Cloudflare returns non-ISO 2-letter codes:
   - `T1` ‚Üí Tor exit node ‚Üí üîí "Tor"
   - `XX` ‚Üí Unknown ‚Üí üåê "Unknown"
   - `A1` ‚Üí Anonymous proxy ‚Üí üîí "Proxy"
   - `A2` ‚Üí Satellite provider ‚Üí üì° "Satellite"
   - `EU` ‚Üí European Union ‚Üí üá™üá∫ (has a real flag emoji)
   - `AP` ‚Üí Asia/Pacific ‚Üí üåè "Asia-Pacific"
   - Standard codes: `String.fromCodePoint(...[...code].map(c => 0x1F1E6 + c.charCodeAt(0) - 65))`

9. **Empty state**: Blog is new ‚Äî periods may have zero data. Show friendly message, not broken charts or empty tables.

10. **Loading state (FOUC)**: Page loads as static HTML, then fetches `/api/stats`. Use CSS skeleton placeholders (animated gray bars matching final layout dimensions). Pure CSS `@keyframes` shimmer. Replaced by real content when data arrives. No layout shift (good CLS).

11. **esbuild CSS merge**: When `stats.ts` imports both `uplot/dist/uPlot.min.css` and `../styles/stats.css`, esbuild merges them into a single `stats.css` output (verified: 768 bytes gzip total). Don't also add `stats.css` as a separate entry point ‚Äî it would conflict.

## Best Practices

1. **‚úÖ Separate entry point per page type**: `main.ts` for all pages, `stats.ts` for `/stats` only. Templates control which `<script>` tags load. This IS code splitting for SSG ‚Äî no framework needed.

2. **‚úÖ CSS variables as the Canvas theme bridge**: Read `--color-link`, `--color-border`, `--color-text-muted`, `--color-bg` via `getComputedStyle()`. Pass as uPlot `stroke`, `fill`, axis `stroke`, grid `stroke`. On theme change, re-read and recreate.

3. **‚úÖ Percentage-width bars for ranked lists**: Same pattern as Plausible's `bar.js` ‚Äî `<div>` with `width: ${(value / maxValue) * 100}%` and `background: var(--color-surface)`. Pure CSS, no chart library.

4. **‚úÖ URL-based period selector**: `/stats?days=7`, `/stats?days=30`, `/stats?days=90`. Shareable, bookmarkable. Client JS reads `URLSearchParams`, passes to `/api/stats?days=N`. Default: 30.

5. **‚úÖ Country code ‚Üí flag emoji with fallback map**: Standard ISO codes get flag emoji via `String.fromCodePoint()`. Non-standard codes (T1, XX, A1, A2, AP) get a lookup table with descriptive emoji + label.

6. **‚úÖ Skeleton loading state**: CSS-only `@keyframes` shimmer on placeholder elements matching final layout. Replaced when data arrives. No layout shift.

7. **‚úÖ `MutationObserver` for theme changes**: Decoupled from `@nisli/core` signals. Stats module watches `data-theme` attribute on `<html>`. Framework-independent.

8. **‚úÖ Fixed chart height, responsive width**: Height 300px (fixed). Width from `container.clientWidth` via `ResizeObserver`. Avoids CSS layout calculation bugs with uPlot.

9. **‚úÖ `pageShell` head param**: Add optional `head?: string` to `pageShell()` for per-page `<link>` and `<script>` tags in `<head>`. Correct HTML semantics, no FOUC, reusable for future pages.

### stats.ts (client) ‚Äî Module Structure

```
1. Imports: uPlot, uPlot CSS, stats.css
2. Types: StatsResponse (matches API shape)
3. Constants: special country codes map (T1‚Üíüîí, XX‚Üíüåê, etc.)
4. getColors(): read CSS vars via getComputedStyle ‚Üí uPlot color config
5. fetchStats(days): GET /api/stats?days=N ‚Üí StatsResponse (with error handling)
6. toUPlotData(by_day): row objects ‚Üí [[timestamps], [views], [visitors]] (seconds, length ‚â• 2)
7. renderTotals(totals): populate 3 number cards, remove skeleton class
8. renderChart(by_day, container): create uPlot, setup ResizeObserver
9. renderList(data, container, labelFn): generic ranked list (pages, referrers, countries)
10. countryLabel(code): flag emoji + name, with fallback map for non-ISO codes
11. init():
    - Read ?days from URLSearchParams (default 30)
    - Highlight active period pill
    - fetchStats ‚Üí render all sections (or show error)
    - Setup MutationObserver on data-theme ‚Üí recreate chart
    - Setup period pill click handlers ‚Üí pushState + re-fetch (no page reload)
    - Setup popstate listener (back button)
```

### Period Selector Behavior

```
Template renders:  <button data-days="7">7d</button> <button data-days="30">30d</button>
                   <button data-days="90">90d</button> <button data-days="3650">All</button>
stats.ts:          Reads ?days from URL, adds 'active' class to matching button
Click handler:     history.pushState({}, '', '?days=N') ‚Üí re-fetch ‚Üí re-render
Back button:       window.addEventListener('popstate', () => re-read URL ‚Üí re-fetch)
Default:           30 days (no ?days param)
"All" period:      days=3650 (API already handles this ‚Äî returns all data)
```

### uPlot Configuration (distilled)

```
Data format:  [[ts1, ts2, ...], [views1, views2, ...], [visitors1, visitors2, ...]]
              Timestamps in unix seconds. Arrays length ‚â• 2 (pad if single day).
Series 0:     x-axis (timestamps) ‚Äî auto-formatted by uPlot as dates
Series 1:     Views ‚Äî area fill (stroke: --color-link, fill: --color-link @ 0.15 opacity)
Series 2:     Visitors ‚Äî line only (stroke: --color-text-muted, no fill)
Axes:         x: date labels. y: integer counts.
Grid:         stroke: --color-border
Cursor:       crosshair enabled, snap to nearest point
              points.fill: explicitly set for dark mode (issue #780)
Legend:        inline, live values on hover
Width:        container.clientWidth (via ResizeObserver)
Height:       300px fixed (avoids flexbox bugs ‚Äî issue #1075)
Theme:        MutationObserver on data-theme ‚Üí getColors() ‚Üí destroy + recreate
```

### Ranked List HTML Pattern (distilled)

```html
<!-- Same visual pattern as Plausible's bar.js -->
<div class="stats-row">
  <div class="stats-bar" style="width: 75%"></div>
  <span class="stats-label">/the-agentic-product-engineer/</span>
```

- Bar: absolute-positioned `<div>`, `width: ${(value / maxValue) * 100}%`, `background: var(--color-surface)`
- Label + value overlaid on top via relative positioning
- Max items: 10 per list (Plausible uses 9)
- Empty state: "No data for this period" message instead of empty table

### Error Handling

- Fetch failure (network, 500, timeout): show "Unable to load stats. Try refreshing." in content area
- Empty data (0 views): show totals as 0, chart as flat line, lists show "No data for this period"
- Single data point: pad `by_day` with a zero-value entry for the previous day (uPlot needs length ‚â• 2)

### CSS Loading Sequence (verified)

```
1. Browser requests /stats/index.html
2. <head> contains: <link href="/main.css"> + <link href="/stats.css"> + <script src="/stats.js" type="module">
3. CSS loads first (render-blocking) ‚Üí skeleton placeholders visible with shimmer animation
4. JS loads (deferred, type="module") ‚Üí fetches /api/stats ‚Üí replaces skeletons with data
5. No FOUC, no layout shift ‚Äî skeleton dimensions match final layout
```
